# gocron v1.6.0 - 更新日志

## 更新概述

本版本主要是对前端依赖和后端环境的现代化升级，解决了很多过时组件的依赖问题。

## 主要更新内容

### 前端依赖更新

- axios 从 ^0.18.0 更新到 ^1.6.0

  - 提供了新的 API 和更好的 TypeScript 支持
  - 改进的错误处理机制

- element-ui 从 ^2.3.6 更新到 ^2.15.13

  - 新增了多个组件和功能
  - 修复了大量的 Bug 和性能问题
  - 提升了国际化支持

- vue 从 ^2.5.2 更新到 ^2.7.16

  - 引入了对 Composition API 的支持 (作为插件)
  - 改进的模板编译器和性能优化
  - 安全修复和稳定性提升

- vue-router 从 ^3.0.1 更新到 ^3.5.4

  - 性能优化和 bug 修复
  - 更好的导航守卫和路由解析功能

- vuex 从 ^3.0.1 更新到 ^3.6.2

  - 更好的 typescript 支持
  - 提供了新的辅助函数
  - 稳定性和安全性提升

- webpack 相关依赖更新

  - webpack 从较低版本更新到 4.47.0
  - 更新了 webpack-dev-server、babel-loader、vue-loader 等相关插件
  - 替换了 ExtractTextWebpackPlugin 为 MiniCssExtractPlugin

- babel 依赖更新

  - 更新了 @babel 相关依赖包
  - 确保了对 ES2015+ 语法的正确转换

- eslint、postcss 等开发依赖更新
  - 确保代码质量和一致性

### 构建工具链修复

- 修复了 webpack 配置，解决与现代依赖的兼容性问题
- 修正了 CopyWebpackPlugin 的配置方法，使用新 API
- 更新了 webpack-merge 导入方式，解决版本不兼容问题
- 解决了开发服务器的启动问题
- 修复了 CSS 提取插件的使用方法

### Go 后端环境

- 解决了 Go 环境的构建和依赖管理问题
- 安装了 statik 工具用于静态资源嵌入
- 修正了 PATH 配置问题，确保命令正常执行

### 安全性改进

- 更新了存在已知安全漏洞的旧版依赖
- 改进了整体项目的安全性和兼容性
- 修复了前端依赖中的潜在安全风险

### 性能优化

- 修复了构建过程中的一些性能问题
- 减少了前端 bundle 大小
- 提升了整体构建效率
